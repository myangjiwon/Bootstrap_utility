1.부트스트랩 - 유틸리티

1.1 API

■ 정의

유틸리티 API는 유틸리티 클래스를 생성하기 위한 Sass 기반 도구입니다.
부트스트랩의 유틸리티 API는 쉽게 스타일을 적용할 수 있습니다.
다양한 값과 옵션을 통해 유연하게 스타일을 조절할 수 있습니다.
반응형 디자인을 위한 클래스들이 포함되어 있어 다양한 화면 크기에 맞게 스타일을 조절할 수 있습니다.
투명도를 조절하는 새로운 유틸리티를 만들어 쓸 수 있습니다.

■ 예시

● SCSS

$utilities: (
"opacity": (
property: opacity,
class: opacity,
state: hover,
values: (
0: 0,
25: .25,
50: .5,
75: .75,
100: 1,
)
)
);

● CSS

.opacity-0-hover:hover { opacity: 0 !important; }
.opacity-25-hover:hover { opacity: .25 !important; }
.opacity-50-hover:hover { opacity: .5 !important; }
.opacity-75-hover:hover { opacity: .75 !important; }
.opacity-100-hover:hover { opacity: 1 !important; }

1.5 디스플레이 (Display)

■ 정의

디스플레이(Display)는 HTML 요소들의 표시 방법을 조정하고 화면에 어떻게 배치할지 결정할 수 있습니다.
쉬운 레이아웃 조작이 가능하며, 반응형 디자인을 쉽게 구현할 수 있고 시각적효과를 손쉽게 구현할 수 있습니다.

■ 예시

# inline

<div class="d-inline p-2 text-bg-primary">d-inline</div>
<div class="d-inline p-2 text-bg-dark">d-inline</div>


# block

<span class="d-block p-2 text-bg-primary">d-block</span> <span class="d-block p-2 text-bg-dark">d-block</span>



* none: 요소를 화면에서 숨깁니다.
* inline: 인라인 요소로 표시되며, 컨텐츠 크기만큼만 영역을 차지합니다.
* inline-block: 인라인 레벨의 속성을 갖지만 블록 레벨과 같은 박스 모델을 가집니다.
* block: 블록 레벨 요소로 표시되며, 가로폭 전체를 차지합니다.
* grid: 부모 요소를 그리드 컨테이너로 만들어 자식 요소들을 그리드 아이템으로 배치할 수 있게 합니다.
* Inline-grid: 인라인 레벨로 표시하면서 동시에 그리드 레이아웃을 사용할 수 있게 해줍니다.
* Table: 테이블(Table) 요소처럼 표시하는 데 사용됩니다.
* table-cell: 테이블 레이아웃과 유사한 방식으로 요소들을 가로로 배열합니다 html의 td 요소와 유사한 방식으로 동작합니다..
* table-row: 수평적인 테이블 행 형태로 배치합니다. html의 tr 요소와 유사한 방식으로 동작합니다.
* flex: 플렉스 컨테이너로 요소를 배치합니다.
* Inline-flex: 인라인 레벨로 표시하면서 동시에 해당 요소를 플렉스 박스 컨테이너로 지정합니다.

반응형 디자인에서 요소를 특정 화면 크기에서만 표시하거나 숨길 수 있습니다.
.d-none 클래스 또는 .d- {sm, md, lg, xl, xxl} -none 클래스 중 하나를 사용하면 됩니다.

플렉스박스를 사용하면 그리드, 열, 네비게이션바, 레이아웃 정렬 및 크기조정을 신속하게 할 수 있습니다.

1.7 플로트 (Float)

■ 정의

플로트는 요소를 좌우 방향으로 띄워서 텍스트나 요소를 감싸거나, 다른 요소와 옆으로 배치하는 데 사용됩니다.

■ 예시

Float start on all viewport sizes

Float end on all viewport sizes

Don't float on all viewport sizes

<div class="float-start">Float start on all viewport sizes</div>
<div class="float-end">Float end on all viewport sizes</div>
<div class="float-none">Don't float on all viewport sizes</div>

.float-start: 요소를 왼쪽으로 플로팅합니다.
.float-end: 요소를 오른쪽으로 플로팅합니다.
.clearfix: floating 된 요소들의 높이를 정상화하기 위해 사용됩니다. 부모 요소가 자식 요소의 높이를 올바르게 계산하여 레이아웃을 제대로 유지할 수 있습니다.


1.8 상호작용 (Interaction)

■ 정의

사용자가 웹 사이트의 콘텐츠와 상호 작용하는 방식을 변경하는 유틸리티 클래스입니다.

■ 예시

* 텍스트 선택 사용자가 콘텐츠와 상호 작용할 때 콘텐츠가 선택되는 방식을 변경합니다.

<p class="user-select-all">This paragraph will be entirely selected when clicked by the user.</p>
<p class="user-select-auto">This paragraph has default select behavior.</p>
<p class="user-select-none">This paragraph will not be selectable when clicked by the user.</p>

* 포인터 이벤트
.pe-none: 해당 요소의 포인터 이벤트를 비활성화합니다. 이 요소 위에서의 마우스 이벤트가 무시됩니다.
사용자가 해당 요소를 클릭하거나 호버할 때 효과가 없도록 만들어줍니다.
.pe-auto: 이 클래스는 요소에 대한 기본 포인터 이벤트를 복원합니다. 이전에 비활성화되었던 포인터 이벤트를 다시 활성화하여 마우스 이벤트가 해당 요소에 올바르게 작동하도록 합니다.

이 요소는 마진과 패딩이 각각 적용되어 있습니다.

<div class="m-3 p-4 bg-primary"> <p class="m-2 p-3 bg-secondary">

* Margin 클래스: .m-{breakpoint}-{sides}-{size}: 여백을 지정합니다.
여백의 크기는 {size}로 조절하며, {sides}는 t (상단), b (하단), l (왼쪽), r (오른쪽), x (좌우), y (상하), a (모든 방향) 중 선택할 수 있습니다.
예: .mb-3 (하단 여백), .mx-4 (좌우 여백)

* Padding 클래스: .p-{breakpoint}-{sides}-{size}: 패딩을 지정합니다.
패딩의 크기는 {size}로 조절하며, {sides}는 t (상단), b (하단), l (왼쪽), r (오른쪽), x (좌우), y (상하), a (모든 방향) 중 선택할 수 있습니다.
예: .py-2 (상하 패딩), .pl-5 (왼쪽 패딩)


1.17 텍스트 (Text)

■ 용도

정렬, 줄바꿈, 굵기 등을 제어하는 ​​일반적인 텍스트 유틸리티입니다.

■ 예시

This text has a line underneath it.

This text has a line going through it.

This link has its text decoration removed

<p class="text-decoration-underline">This text has a line underneath it.</p>
<p class="text-decoration-line-through">This text has a line going through it.</p>
<a href="#" class="text-decoration-none">This link has its text decoration removed</a>


1.18 수직정렬 (Vertical alignment)

■ 용도

인라인, 인라인 블록, 인라인 테이블 및 테이블 셀 요소의 수직 정렬을 쉽게 변경할 수 있습니다.

■ 예시

baseline top middle bottom text-top text-bottom

<span class="align-baseline">baseline</span>
<span class="align-top">top</span>
<span class="align-middle">middle</span>
<span class="align-bottom">bottom</span>
<span class="align-text-top">text-top</span>
<span class="align-text-bottom">text-bottom</span>

1.19 가시성 (Visibility)

■ 용도

가시성 유틸리티를 사용하여 표시를 수정하지 않고 요소의 가시성을 제어합니다.
가시성 유틸리티를 사용하여 요소의 visibility를 설정합니다.
이러한 유틸리티 클래스는 display 값을 전혀 수정하지 않으며 레이아웃에 영향을 미치지 않습니다.
invisible 요소는 여전히 페이지의 공간을 차지합니다.

■ 예시

<div class="visible">...</div>
<div class="invisible">...</div>

1.20 Z-index

■ 정의

요소의 쌓임 순서를 정의합니다. 이것은 주로 포지셔닝된 요소들의 배치 순서를 결정할 때 사용됩니다. 더 높은 Z-index 값을 가진 요소가 더 위로 쌓이게 됩니다.

■ 예시

z-3
z-2
z-1
z-0
z-n1



일반적으로 z-index 값이 높은 요소가 z-index 값이 낮은 요소 위에 겹쳐지게 됩니다. 따라서 여러 요소가 모두 position: absolute와 함께 z-index 값을 가지고 있다면, z-index 값이 높은(예를 들어 z-3) 요소가 z-index 값이 낮은(예를 들어 z-2, z-1, z-0, z-n1) 요소 위에 위치하게 됩니다.
